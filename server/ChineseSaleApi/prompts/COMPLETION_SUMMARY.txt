## 🎯 PROPOSED ARCHITECTURE

### 6 Microservices

1. **AuthService** - JWT tokens, user registration, password management
2. **GiftService** - Gift catalog CRUD, search, inventory
3. **DonorService** - Donor profiles, email validation
4. **CategoryService** - Gift taxonomy, categories
5. **OrdersService** - Shopping carts, checkout orchestration (NEW)
6. **PurchaseService** - Tickets, sales reporting, revenue (NEW)

### Technology Stack

- **Framework:** ASP.NET Core 8 / C# 12
- **Databases:** SQL Server (6 instances, one per service)
- **Cache:** Redis
- **Message Broker:** RabbitMQ 4.0+
- **API Gateway:** Kong 3.4
- **Orchestration:** Kubernetes
- **Monitoring:** Prometheus + Grafana
- **CI/CD:** GitHub Actions

### Migration Pattern

- **Strangler Fig** (parallel running, gradual cutover)
- **Duration:** 6-9 months (40 weeks)
- **Phases:** 6 sequential phases
- **Safety:** 1-week rollback period per phase
- **Deployments:** Zero-downtime

## 🏗️ KEY ARCHITECTURAL DECISIONS

### Database Strategy

- **Database-per-Service:** 6 separate SQL Server databases
- **Eventual Consistency:** Accept 30-60 second data delays
- **Saga Pattern:** Distributed transaction coordination
- **Event-Driven:** RabbitMQ for async communication
- **Data Migration:** Dual-write pattern with reconciliation

### Communication

- **Synchronous (REST):** Real-time validation with circuit breakers
- **Asynchronous (Events):** Decoupled services via RabbitMQ
- **Sagas:** Complex workflows spanning multiple services

### Security

- **JWT Tokens:** 1 hour expiry + 7 day refresh token
- **Token Revocation:** Blacklist in Redis
- **RBAC:** Role-Based Access Control
- **Secret Management:** HashiCorp Vault
- **API Gateway:** Kong with security plugins

### Deployment

- **Containerization:** Docker per service (multi-stage builds)
- **Orchestration:** Kubernetes with HPA (auto-scaling)
- **CI/CD:** GitHub Actions with multi-service matrix
- **Configuration:** Helm for K8s package management
- **Health Checks:** Liveness & readiness probes
